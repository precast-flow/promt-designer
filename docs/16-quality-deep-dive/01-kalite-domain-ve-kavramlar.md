# Kalite domain ve kavramlar

## Temel nesneler

| Kavram | Açıklama |
|--------|----------|
| **Numune** | Belirli bir beton / döküm / tarih için alınan örnek; kimlik ve etiket |
| **Test isteği** | Hangi testlerin yapılacağı (slump, basınç, …) |
| **Test sonucu** | Ölçüm değerleri, birim, cihaz, operatör, tarih |
| **Test paketi** | Bir numuneye bağlı birden fazla test satırı |
| **Beton tanımı / reçete** | Onaylı karışım (üretim tarafında seçilen ile aynı havuz) |
| **Kalite raporu** | Şablonlu çıktı; firma logosu, imza alanları, ekler |
| **Teslim kaydı** | Raporun kime, ne zaman, hangi kanalla iletildiği (mock) |

## Durum makinesi (özet)

Numune / test paketi: `Alındı → Testte → Sonuç girildi → İncelendi → Onaylandı / Reddedildi → Raporlandı`.

## Çoklu bağlam

Her kayıt için zorunlu bağlam: **Firma**, **Fabrika** (varsa), **Proje** (opsiyonel), **Üretim emri / döküm referansı** (opsiyonel ama izlenebilirlik için önerilir).
